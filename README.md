# yakyu-coach-ai

## サービス概要
野球中継の専門用語についていけなかったり、子どもの試合で何が起きているのか分からない人のためのアプリです。

「あの走るやつ」「ダブルなんとか」など曖昧な検索でも、AIがあなたの理解度に合わせて、野球以外の例えや観戦ポイントを交えて分かりやすく解説します。

調べるだけでその場で理解でき、恥ずかしくて聞けない疑問もすぐに解決できます。

## このサービスへの思い・作りたい理由
高校時代、野球部の保護者会に父が参加しましたが、周りは野球経験者や指導者ばかりで、野球のルールしか分からない父は戦術や専門用語の話についていけず、あまり会話に参加できなかったみたいです。当時は「仕方ない」と思っていましたが、自分が社会人になり野球未経験者と話す中で、野球に興味はあるのに理解できず楽しめない人が多いと気づきました。野球観戦は好きだけど、詳しい話はわからない人、子どもが野球を始めて、もっと理解したいと思っている保護者、野球に興味はあるけど、どこから学べばいいかわからない人でも理解度に合わせて学べるアプリがあれば、もっと野球を楽しめるのではと思い、開発を決めました。

## ユーザー層について
### 野球初心者
- パートナーが野球好きで一緒に観戦したい人
- 野球中継の専門用語が分からない方
- 職場の野球談義についていけない方
- 野球部のマネージャー

### 子供が野球を始めた保護者
- 少年野球・部活動の保護者
- 試合をもっと楽しみながら応援したい方
- 子供に野球を教えたい親

## サービスの利用イメージ
- 試合観戦中にリアルタイムで用語検索
- 野球ニュースを見ている時
- 野球談議中にわからない用語が出てきた時

### 具体的な使い方の流れ
1. 初回利用時に理解度（初心者/中級者/上級者）を選択
2. 分からない用語を検索
3. 2-3秒で理解度に合わせた説明が表示
4. 関連用語3つが提案され、自然に学習を促進
5. 理解が進んだら設定で理解度をアップデート(初心者→中級者)

## サービスの差別化ポイント・推しポイント
このアプリの最大の特徴は、AIがユーザーの理解度に合わせて最適な説明を行ってくれる点です。
野球の世界では、同じ意味でも異なる言い回しや略語が非常に多く、初心者にとって理解が難しい場面が多々あります。
（例：盗塁＝スチール、ダブルプレー＝ゲッツー、犠牲バント＝送りバントなど）

既存の辞書アプリでは、正式名称を覚えていないと目的の言葉が見つけられなかったり、別の呼び方を知らないために検索できないことがあります
一方、AI　であれば
- 略語や俗語にも対応できます。
- 「あの走るやつ」など曖昧な検索も可能
- 実際の観戦や会話で使われる自然な言い回しも理解してくれる といった特徴があり、リアルな観戦・会話シーンで強さを発揮します。

 さらに、ユーザーの理解度に応じて説明の仕方を調整します。
 - 初心者：野球以外の例えを使い、専門用語を使わない優しい説明
 - 中級者：観戦時の注目ポイントや状況説明を追加
 - 上級者：戦術背景や配球意図、など高度な解説
 これにより、初心者から野球経験者まで、誰でも心地よく理解できる体験を提供できます.

 また、検索した用語に加えて関連用語を3つ自動で提示し、 知識が“点”から“線”につながるようになり、自然に野球の理解を深めることができるのではないかなと思っています。

## 機能候補
MVPリリース
- 初回の理解度選択 [ 初心者 ] [ 中級者 ] [ 上級者 ]
- openAIのAPIを使用し、ユーザーの理解度 (初心者 中級者 上級者) にわけて、結果を出力
- ユーザーの理解度を変更するためのボタンを設置( 初心者⇄中級者⇄上級者)
- 理解度設定をローカルストレージに保存し、次回以降も維持 (別端末・ブラウザでは再設定が必要)

**理解度の基準**
- 初心者 ほとんどない / ルールもよくわからない
- 中級者 観戦やゲーム経験はある / 大まかなルールはわかる
- 上級者 プレー経験あり / 戦術も興味あり

**検索結果画面の構成**
検索結果は、選択した理解度に応じて表示内容が変化します。

| 理解度 | 表示内容 |
|:--|:--|
| 初心者 | 説明文（専門用語を使わない）＋ 例え（野球以外の身近な例） |
| 中級者 | 説明文（やや専門的）＋ 観戦ポイント（どこを注目すると良いか） |
| 上級者 | 説明文（専門的）＋ 戦術背景（配球意図・チーム戦略など） |
| 共通 | 関連用語を3つ自動表示（タップで再検索） |

本リリース
- スマホでも見れるようにレスポンシブ対応
- 関連用語3つの提案（継続利用促進）
- 人気用語キャッシュ機能
  - DB保存 → 即時表示
  - API呼び出し回数の最適化
